## What is Modules

Modules is the way of dismantling your code in various small blocks and storing those small block in various files, which are called modules. So basically a module, library and packages are the same things, all they do is import a functionality from other file, that file may be locally available on your machine which is generated by you or that file may be available on interest that is generated by someone else.

## Node JS Architecture

Request -> BACKEND -> Event Queue -> Event Loop

Non-Blocking Operations / Asynchronous Tasks -> Process -> Response

Blocking Operations / Synchronous Tasks -> Thread Pool -> Assign Worker (if available) -> Return Result -> Response

Thread ---- Worker
Pool ---- Bunch

Default Thread Pool Size = 4 ---- Max Depends on Hosting Server Virtual CPUs / Cores

The Event Queue and Event Loop live inside the Node.js process memory on your hosting server (e.g., AWS, DigitalOcean, or your local machine during development). Furthermore, the Event Queue is actually managed by libuv (a C++ library built into Node.js), while the V8 engine is simply responsible for executing the JavaScript code once the queue hands it over.

### Example

1. The Click Triggers an HTTP RequestWhen the user clicks, their browser sends a digital packet (an HTTP Request) across the internet. This packet travels from their device straight to Vercel's closest data center.

2. Vercel Wakes Up Your Node.js Code. Vercel receives that request and instantly spins up a small, temporary Node.js container. Inside this fresh container, the Event Queue is created in memory.

3. The Code Enters the Event QueueYour server-side code (like a Next.js API route or a Vercel Function) starts running: The Trigger: The incoming click request is placed into the Event Queue.The Execution: The Event Loop grabs that request from the queue and hands it to the V8 Engine to process your JavaScript logic (e.g., fetching data).The Response: Your code generates a response (like JSON data or a new webpage) and sends it back to the user's browser.

4. The Queue Disappears The moment the browser receives the response, Vercel shuts down or freezes the container. The Event Queue is completely wiped out until the next click happens.

## HTTP Methods

**Get**: Get is used to retrieve data from the server. It is a read-only operation and does not modify any data on the server.

**POST**: Post is used to send data to the server. It is used to create new resources on the server.

**PUT**: Put is used to update existing resources on the server. It replaces the entire resource with the new data provided.

**PATCH**: Patch is used to update existing resources on the server. It only modifies the specified fields of the resource.

**DELETE**: Delete is used to remove resources from the server. It deletes the specified resource.
